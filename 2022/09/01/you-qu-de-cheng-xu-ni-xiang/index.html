有趣的程序逆向

#include<stdio.h>
int max(int a,int b)
{
  int z;
  if(a>b) z=a;
  else z=b; 
  //return (z);    
} 
int main()
{
  int a,b,c;
  scanf("%d %d",&a,&b);
  c=max(a,b);
  printf("max = %d\n",c);
  return 0;
}

将此程序用devc++进行编译，一定得是devc++

max函数并没有返回值并且devc++上竟然能成功编译，最离谱的是大小比较功能还在，max函数没有return回来最大值为什么主函数最后能输出最大值，拖入ida进行分析

首先是主函数，可以看到

输入的值放入了[rbp+var_C]和[rbp+var_8]中，接着把输入的值放入edx和ecx中，然后通过call指令进入max函数

这里最终输入的值在eax和[rbp+arg_8]中，并通过cmp进行比较，如果eax的值小于等于[rbp+arg_8],则通过jle跳转，这里放大跳转的loc_40154E

将[rbp+arg_8]存储的值放入eax,即将较大的那个书放入eax中，回到上面,若在cmp比较时eax的值大于[rbp+arg_8]则继续向下执行，经过了几次mov，eax的值并没有发生变化，接着返回主函数

可以发现最终输出的值就是eax存储的值，即在max函数中的较大值。
